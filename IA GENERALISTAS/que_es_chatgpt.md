# Qué es ChatGPT, qué puede hacer y qué no puede hacer

---

## 1. Idea rápida: ChatGPT en lenguaje de calle

ChatGPT es una aplicación con la que puedes conversar por escrito, por voz o con archivos para pedir ayuda con tareas de información, redacción, análisis, aprendizaje, programación, creatividad o planificación.

Dicho de forma sencilla: es como tener un asistente digital al que puedes pedirle que te explique algo, te ayude a escribir, revise un texto, compare opciones, genere ideas, analice documentos o te acompañe paso a paso en una tarea.

Pero hay una idea muy importante:

**ChatGPT no es una persona, no piensa como una persona y no “sabe” las cosas como las sabe un experto humano.**

Funciona generando respuestas probables a partir de patrones aprendidos, instrucciones recibidas y, según el caso, herramientas conectadas como búsqueda web, análisis de archivos, generación de imágenes o ejecución de código.

---

## 2. La confusión típica: ChatGPT no es exactamente “el modelo”

En formación conviene separar bien estos conceptos:

| Concepto | Explicación sencilla |
|---|---|
| **IA** | Campo general que busca crear sistemas capaces de realizar tareas que normalmente asociamos a inteligencia humana. |
| **IA generativa** | Tipo de IA capaz de generar contenido nuevo: texto, imágenes, código, resúmenes, ideas, esquemas, etc. |
| **Modelo de lenguaje** | Sistema entrenado para procesar y generar lenguaje. Predice y construye texto a partir del contexto. |
| **LLM** | “Large Language Model”. Modelo de lenguaje grande, entrenado con muchísimos datos y capaz de realizar tareas lingüísticas complejas. |
| **GPT** | Familia de modelos de OpenAI. GPT significa “Generative Pre-trained Transformer”. |
| **ChatGPT** | Aplicación/interfaz conversacional que permite usar modelos y herramientas de OpenAI. |
| **Chatbot** | Programa que conversa con usuarios. Puede ser simple o muy avanzado. |
| **Agente** | Sistema que, además de responder, puede planificar pasos, usar herramientas y ejecutar acciones con cierto grado de autonomía controlada. |

La frase correcta sería:

> “ChatGPT es una aplicación que usa modelos de IA, como modelos GPT, para mantener conversaciones y ayudar en tareas.”

No sería preciso decir siempre:

> “ChatGPT es el modelo.”

Porque ChatGPT puede cambiar de modelo, usar diferentes modos de razonamiento y apoyarse en herramientas externas.

## 3. Qué puede hacer ChatGPT

ChatGPT puede ayudar en muchas tareas, especialmente cuando hay lenguaje, información o razonamiento de por medio.

### 3.1 Explicar conceptos

Puede explicar temas complejos con distintos niveles de dificultad.

Ejemplos:

- “Explícame qué es una API como si tuviera 12 años.”
- “Explícame Docker para alguien que viene de ingeniería industrial.”
- “Dame una explicación técnica de Kubernetes con ejemplos operativos.”

Una ventaja importante es que puedes pedirle que adapte el nivel:

- más simple,
- más técnico,
- con ejemplos,
- con analogías,
- con pasos prácticos,
- con errores frecuentes.

---

### 3.2 Redactar y mejorar textos

Puede ayudarte a escribir, revisar o adaptar textos:

- correos,
- informes,
- propuestas,
- publicaciones de LinkedIn,
- documentación técnica,
- guiones de clase,
- textos comerciales,
- currículums,
- instrucciones de uso.

Ejemplo:

```text
Reescribe este correo para que suene profesional, claro y directo, sin parecer agresivo.
```

También puede transformar un texto según el público:

```text
Adapta este texto técnico para alumnos sin conocimientos de informática.
```

---

### 3.3 Resumir información

Puede resumir textos largos, extraer ideas clave o convertir contenido desordenado en una estructura clara.

Puede generar:

- resumen ejecutivo,
- lista de puntos clave,
- tabla comparativa,
- esquema para presentación,
- preguntas de examen,
- glosario,
- checklist operativo.

Ejemplo:

```text
Resume este documento en 10 puntos y separa hechos, riesgos y decisiones pendientes.
```

---

### 3.4 Ayudar a aprender

Puede actuar como tutor interactivo.

No solo responde: también puede preguntar, corregir, proponer ejercicios y explicar errores.

Ejemplos:

```text
Enséñame Git desde cero con ejercicios prácticos.
```

```text
Hazme preguntas tipo test sobre IA generativa y corrige mis respuestas.
```

```text
Explícame este error de Python sin asumir que sé programar.
```

---

### 3.5 Programar y revisar código

ChatGPT puede ayudar con:

- escribir código,
- explicar código existente,
- detectar errores,
- proponer refactors,
- crear tests,
- documentar funciones,
- generar scripts sencillos,
- explicar mensajes de error.

Ejemplo:

```text
Este código de FastAPI falla. Explícame la causa probable, el cambio mínimo y cómo probarlo.
```

Pero hay que tener cuidado: que el código “parezca correcto” no significa que sea seguro, eficiente o apto para producción.

En programación conviene pedir siempre:

- supuestos,
- riesgos,
- pruebas,
- casos límite,
- impacto en seguridad,
- impacto en mantenimiento,
- cómo desplegarlo o revertirlo.

---

### 3.6 Analizar archivos

Según el plan, la configuración y las herramientas disponibles, ChatGPT puede trabajar con archivos como:

- documentos de texto,
- PDF,
- hojas de cálculo,
- imágenes,
- presentaciones,
- fragmentos de código,
- datos tabulares.

Puede ayudar a:

- resumir un PDF,
- revisar una presentación,
- detectar incoherencias,
- extraer tablas,
- analizar datos,
- generar gráficos,
- encontrar errores en un documento.

Limitación importante: si el archivo está mal escaneado, incompleto, bloqueado o contiene información ambigua, el análisis puede fallar o ser parcial.

---

### 3.7 Crear imágenes o ayudar con contenido visual

Cuando tiene herramientas de imagen disponibles, ChatGPT puede ayudar a generar imágenes, adaptar estilos visuales, crear ideas para infografías, proponer layouts o redactar prompts para herramientas visuales.

Ejemplos:

```text
Crea una imagen para explicar qué es un agente de IA, sin texto, estilo tecnológico sobrio.
```

```text
Dame una estructura visual para una diapositiva sobre IA generativa en pymes.
```

No obstante, las imágenes generadas pueden contener errores visuales, texto mal escrito, detalles inventados o composiciones poco realistas.

---

### 3.8 Ayudar a tomar decisiones

ChatGPT puede ordenar información y comparar alternativas.

Puede ayudar a construir:

- matrices de decisión,
- análisis de pros y contras,
- criterios de evaluación,
- riesgos,
- planes de acción,
- rutas de aprendizaje,
- estimaciones iniciales.

Ejemplo:

```text
Compara estas tres opciones para desplegar una aplicación: servidor local, Railway y AWS. Incluye coste, seguridad, debugging y dependencia del proveedor.
```

Pero no debe sustituir el criterio profesional en decisiones críticas.

---

## 4. Qué no puede hacer ChatGPT

Tan importante como saber lo que puede hacer es entender sus límites.

---

### 4.1 No garantiza la verdad

ChatGPT puede equivocarse.

A veces genera una respuesta convincente pero falsa. A esto se le suele llamar **alucinación**.

Ejemplos de errores posibles:

- inventar una fuente,
- mezclar conceptos,
- dar una fecha incorrecta,
- explicar una ley desactualizada,
- proponer una librería que no existe,
- afirmar que un archivo dice algo que realmente no dice,
- dar una solución técnica que no funciona al probarla.

Regla práctica:

> Cuanto más importante sea la decisión, más hay que verificar la respuesta.

---

### 4.2 No entiende el mundo como una persona

Aunque pueda hablar de forma natural, ChatGPT no tiene experiencia real, conciencia, intención ni comprensión humana.

No “vive” lo que explica. Opera con patrones, contexto e instrucciones.

Esto no lo hace inútil. Lo hace diferente.

Es muy potente como asistente, copiloto o herramienta de análisis, pero no debe confundirse con un experto humano responsable de la decisión final.

---

### 4.3 No sabe automáticamente todo lo actual

Dependiendo del modelo, configuración y herramientas activas, ChatGPT puede no tener información actualizada.

Si no tiene búsqueda web o acceso a una fuente reciente, puede responder con información antigua.

Ejemplos donde hay que verificar actualidad:

- precios,
- leyes,
- ayudas públicas,
- versiones de software,
- modelos disponibles,
- límites de planes,
- documentación de APIs,
- requisitos administrativos,
- noticias,
- normativas técnicas.

---

### 4.4 No puede acceder a tus sistemas salvo que le des acceso

ChatGPT no puede ver tu ordenador, tus archivos, tu correo, tu calendario, tu empresa o tu repositorio privado salvo que:

- subas archivos,
- pegues información,
- conectes una integración,
- autorices una herramienta,
- ejecutes tú los comandos que te propone,
- trabajes dentro de un entorno que sí tenga acceso.

Esto es importante para seguridad.

No conviene pegar credenciales, contraseñas, tokens, claves API, información personal sensible o datos confidenciales sin una política clara.

---

### 4.5 No convierte automáticamente un prototipo en producción

ChatGPT puede ayudarte a crear una demo, un script o una web sencilla.

Pero producción exige más cosas:

- autenticación,
- control de errores,
- logs,
- métricas,
- backups,
- despliegue reproducible,
- pruebas,
- seguridad,
- control de versiones,
- monitorización,
- costes,
- mantenimiento,
- trazabilidad.

Una respuesta generada por IA puede ser útil como punto de partida, pero no sustituye una revisión técnica.

---

### 4.6 No es una autoridad médica, legal, financiera o de seguridad

Puede explicar conceptos, preparar preguntas o ayudarte a entender documentos, pero no debe sustituir a profesionales cualificados en decisiones críticas.

En temas de alto impacto hay que verificar con fuentes oficiales o expertos acreditados.

## 5. Cómo funciona por dentro, explicado progresivamente

---

## 5.1 Nivel 1: el autocompletado llevado al extremo

Una forma sencilla de entenderlo:

> ChatGPT predice qué respuesta tiene más sentido a partir de lo que le escribes.

No copia simplemente una respuesta cerrada de una base de datos. Genera texto nuevo combinando patrones aprendidos durante el entrenamiento y el contexto actual de la conversación.

Por eso puede responder preguntas muy variadas.

También por eso puede equivocarse: está generando una respuesta probable, no consultando siempre una verdad garantizada.

---

## 5.2 Nivel 2: tokens

ChatGPT no procesa exactamente palabras como las vemos las personas.

Procesa unidades llamadas **tokens**.

Un token puede ser:

- una palabra,
- parte de una palabra,
- un signo,
- un fragmento de texto.

Ejemplo aproximado:

```text
ChatGPT puede ayudar a redactar informes.
```

El modelo no lo ve exactamente como una frase humana, sino como una secuencia de piezas internas.

Esto importa porque:

- los modelos tienen límites de contexto,
- las conversaciones largas consumen más tokens,
- los archivos grandes pueden requerir selección o resumen,
- el coste en APIs suele depender de tokens de entrada y salida.

---

## 5.3 Nivel 3: contexto

El **contexto** es todo lo que el modelo recibe para generar una respuesta:

- tu mensaje actual,
- parte de la conversación anterior,
- instrucciones del sistema,
- archivos aportados,
- resultados de herramientas,
- datos recuperados de una base documental,
- preferencias configuradas.

El modelo no responde en el vacío. Responde según el contexto disponible.

Por eso un buen prompt mejora mucho la calidad de la respuesta.

---

## 5.4 Nivel 4: prompt

Un **prompt** es la instrucción que le das.

Un prompt débil sería:

```text
Explícame IA.
```

Un prompt más útil sería:

```text
Explícame qué es la IA generativa para alumnos sin conocimientos técnicos. Usa ejemplos de oficina, evita tecnicismos al principio y termina con una tabla de conceptos clave.
```

Un buen prompt suele incluir:

- objetivo,
- público,
- contexto,
- formato de salida,
- restricciones,
- nivel técnico,
- ejemplos,
- criterios de calidad.

---

## 5.5 Nivel 5: modelo, aplicación y herramientas

En un uso moderno, no todo lo hace “el modelo” directamente.

Hay varias capas:

```text
Usuario
  ↓
Aplicación ChatGPT
  ↓
Modelo de IA
  ↓
Herramientas opcionales
  - búsqueda web
  - análisis de datos
  - generación de imágenes
  - lectura de archivos
  - conectores
  - ejecución de código
```

Esto significa que ChatGPT puede comportarse de forma distinta según:

- el modelo seleccionado,
- el plan del usuario,
- las herramientas disponibles,
- las políticas de seguridad,
- el tipo de archivo,
- la configuración de privacidad,
- el contexto de la conversación.

- Corregir redacciones.
- Aprender programación paso a paso.
- Crear mapas conceptuales.

- Diseñar casos prácticos.
- Preparar presentaciones.
- Crear bancos de preguntas.
- Corregir borradores de materiales.

Diseña una actividad de 45 minutos para explicar ChatGPT a alumnos sin conocimientos técnicos, con dinámica práctica y cierre evaluable.

## 7. Buenas prácticas para usar ChatGPT

---

### 7.1 Dar contexto

Malo:

```text
Hazme un resumen.
```

Mejor:

```text
Hazme un resumen para una presentación de 5 minutos dirigida a alumnos sin conocimientos técnicos. Mantén solo las ideas importantes y añade un ejemplo sencillo.
```

---

### 7.2 Pedir formato

Puedes pedir:

- tabla,
- checklist,
- pasos numerados,
- esquema,
- JSON,
- Markdown,
- correo,
- guion,
- rúbrica,
- plan de clase,
- prompt para otra herramienta.

Ejemplo:

```text
Devuélvelo en Markdown, con títulos H2, tablas y una sección final de errores frecuentes.
```

---

### 7.3 Pedir límites y riesgos

Una buena práctica es no pedir solo la respuesta, sino también sus límites.

Ejemplo:

```text
Dame la respuesta, pero incluye supuestos, riesgos, errores frecuentes y cómo verificarlo.
```

---

### 7.4 Iterar

La primera respuesta no tiene por qué ser la definitiva.

Puedes seguir con:

```text
Hazlo más claro.
```

```text
Baja el nivel técnico.
```

```text
Añade ejemplos reales.
```

```text
Convierte esto en una actividad práctica.
```

```text
Ahora hazlo más profesional.
```

---

### 7.5 Verificar

Para tareas importantes, verifica:

- fuentes,
- fechas,
- normativa,
- cálculos,
- código,
- seguridad,
- compatibilidad,
- licencias,
- datos sensibles.

Regla útil:

> ChatGPT acelera el trabajo, pero no elimina la responsabilidad de revisar.

## 8. Errores frecuentes al usar ChatGPT

| Error | Problema | Mejor enfoque |
|---|---|---|
| Pedir cosas demasiado vagas | La respuesta sale genérica | Dar contexto, público y formato |
| Creer todo sin verificar | Puede haber errores | Comprobar fuentes y pruebas |
| Pegar datos sensibles | Riesgo de privacidad | Anonimizar o usar entornos adecuados |
| Usarlo como buscador perfecto | Puede no estar actualizado | Pedir fuentes o usar búsqueda verificada |
| Copiar código sin probar | Puede fallar o ser inseguro | Ejecutar tests y revisar logs |
| Pedir “hazme una app” sin requisitos | Resultado frágil | Definir alcance, datos, usuarios y despliegue |
| Confundir demo con producción | Faltan seguridad y operación | Añadir observabilidad, tests y despliegue reproducible |

## 9. ChatGPT en prototipo vs producción

Esta distinción es crítica.

---

### 9.1 Uso como prototipo

Un prototipo sirve para probar una idea rápido.

Ejemplos:

- una demo de chatbot,
- un script para procesar documentos,
- una web sencilla,
- un asistente para preguntas frecuentes,
- una prueba de extracción de datos.

En prototipo se acepta cierto margen de error porque el objetivo es aprender.

---

### 9.2 Uso en producción

Producción significa que el sistema afecta a usuarios reales, datos reales, decisiones reales o procesos de negocio.

Aquí hacen falta controles:

- autenticación,
- autorización,
- logs estructurados,
- métricas,
- trazas,
- control de costes,
- gestión de errores,
- límites de uso,
- evaluación de respuestas,
- revisión humana en tareas críticas,
- protección de datos,
- pruebas automáticas,
- rollback,
- monitorización.

---

### 9.3 Superficie de fallo

Cuando metes ChatGPT o un LLM en un sistema, pueden fallar varias capas:

| Capa | Posible fallo |
|---|---|
| Usuario | Pregunta ambigua o mal formulada |
| Prompt | Instrucción incompleta o contradictoria |
| Modelo | Respuesta falsa, incompleta o inconsistente |
| Herramienta | Búsqueda, archivo o conector falla |
| Datos | Fuente desactualizada o mal estructurada |
| API | Timeout, error, límite de uso o coste alto |
| Aplicación | Bug en backend o frontend |
| Seguridad | Exposición de datos o permisos incorrectos |
| Observabilidad | No queda registro suficiente para depurar |

Por eso, en sistemas serios no basta con “poner ChatGPT”. Hay que diseñar el proceso completo.

## 10. Observabilidad mínima en sistemas con IA

Si se usa ChatGPT o un modelo mediante API en una aplicación, conviene registrar información mínima por ejecución:

- quién hizo la petición,
- cuándo,
- con qué modelo,
- con qué prompt o versión de prompt,
- cuántos tokens se usaron,
- cuánto tardó,
- cuánto costó aproximadamente,
- qué documentos se usaron como evidencia,
- qué herramientas se llamaron,
- si hubo error,
- si se aplicó fallback,
- qué respuesta se devolvió.

Ejemplo de traza simplificada:

```json
{
  "trace_id": "abc123",
  "timestamp": "2026-05-28T08:00:00Z",
  "user_id": "user_01",
  "provider": "openai",
  "model": "modelo_usado",
  "input_tokens": 850,
  "output_tokens": 220,
  "latency_ms": 1800,
  "estimated_cost": 0.004,
  "retrieval_status": "EVIDENCE_FOUND",
  "status": "ok",
  "fallback_used": false
}
```

Esto permite responder preguntas operativas:

- ¿Qué falló?
- ¿Cuánto costó?
- ¿Qué modelo respondió?
- ¿Con qué evidencia?
- ¿Cuánto tardó?
- ¿La respuesta fue verificable?
- ¿Se puede reproducir el problema?

---

## 11. Privacidad y seguridad

Antes de usar ChatGPT con información sensible, conviene hacerse estas preguntas:

1. ¿Estoy pegando datos personales?
2. ¿Estoy compartiendo información confidencial de empresa?
3. ¿Hay contratos, credenciales, tokens o claves API?
4. ¿La cuenta es personal, educativa o empresarial?
5. ¿Qué configuración de privacidad está activa?
6. ¿Se permite usar esta herramienta en mi organización?
7. ¿Hay que anonimizar los datos?

Buenas prácticas:

- no pegar contraseñas,
- no pegar claves API,
- no pegar tokens,
- anonimizar nombres y datos personales,
- revisar políticas internas,
- usar cuentas corporativas cuando proceda,
- limitar el acceso a conectores,
- no subir documentación sensible sin permiso.

---

## 12. Diferencia entre ChatGPT, API y un chatbot propio

| Opción | Qué es | Uso típico |
|---|---|---|
| **ChatGPT** | Aplicación lista para usar | Conversar, trabajar con archivos, aprender, redactar, analizar |
| **API de OpenAI** | Interfaz para integrar modelos en software propio | Crear apps, automatizaciones, backends, productos |
| **Chatbot propio** | Aplicación creada por una empresa o persona usando modelos y datos propios | Atención al cliente, soporte interno, búsqueda documental |
| **Agente** | Sistema que usa modelo + herramientas + planificación | Ejecutar tareas compuestas, investigar, operar flujos con supervisión |

Ejemplo:

- Usar ChatGPT: abrir la web y pedir ayuda.
- Usar API: programar una aplicación que llama a un modelo.
- Crear chatbot propio: construir una interfaz conectada a tus documentos.
- Crear agente: permitir que el sistema planifique pasos, consulte herramientas y ejecute acciones controladas.

## 13. Ejemplos de prompts útiles

### Prompt para aprender

```text
Explícame [tema] desde cero. Primero usa lenguaje coloquial, después introduce los conceptos técnicos mínimos. Termina con una tabla de resumen y tres preguntas para comprobar si lo he entendido.
```

### Prompt para revisar un texto

```text
Revisa este texto. Corrige claridad, estructura y precisión. No cambies el significado. Devuélveme una versión mejorada y una lista breve de cambios realizados.
```

### Prompt para analizar un documento

```text
Analiza este documento. Extrae objetivo, ideas clave, riesgos, contradicciones y acciones recomendadas. Si algo no aparece en el documento, indícalo claramente.
```

### Prompt para código

```text
Revisa este código como ingeniero backend. Busca errores, problemas de seguridad, falta de logs, acoplamiento excesivo y problemas de despliegue. Propón cambios mínimos y pruebas.
```

### Prompt para decisiones

```text
Compara estas opciones usando una matriz con criterios: coste, dificultad, riesgo, mantenimiento, seguridad y dependencia del proveedor. Termina con una recomendación razonada.
```

## 14. Resumen ejecutivo

ChatGPT es una aplicación de IA conversacional que permite usar modelos avanzados para generar, transformar, analizar y explicar información.

Puede ser muy útil para aprender, redactar, resumir, programar, analizar documentos, crear ideas y acelerar tareas profesionales.

Pero tiene límites importantes:

- puede equivocarse,
- puede inventar información,
- puede no estar actualizado,
- no sustituye el juicio experto,
- no garantiza seguridad ni producción,
- no debe recibir datos sensibles sin control,
- necesita verificación en tareas importantes.

La forma correcta de usarlo no es como “oráculo”, sino como **copiloto de trabajo**:

> una herramienta potente para acelerar pensamiento, documentación, análisis y ejecución, siempre con revisión humana, contexto adecuado y controles proporcionales al riesgo.

---

## 15. Frase final para clase

ChatGPT no es magia y no es un experto infalible. Es una interfaz muy potente para trabajar con modelos de IA. Bien usado, ahorra tiempo y amplía capacidades. Mal usado, puede producir errores rápidos, bonitos y peligrosamente convincentes.

La clave no es solo “saber preguntarle”. La clave es saber **para qué usarlo, cómo verificarlo y cuándo no delegar en él**.
